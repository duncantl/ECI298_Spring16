Graphics

Various Approaches

We have several different systems for creating plots, and at different levels. We have high-level functions for creating common plots, e.g., histograms, scatterplots, time series plots, etc.

  • graphics package (and others)

  • lattice package

  • ggplot2 package

Some visualization tasks are easiest using the graphics package. It is reasonably straightforward conceptually to customize a plot to anything you want, but you'll be responsible for some of the low-level calculations (e.g., placing the legend, scaling values to a common scale, ensuring axes/scales are the same across plots.)

lattice is quite easy to use for many things and it takes care of several of the things you might have to do yourself in the graphics package plots, e.g, a legend, consistent axes/scales. You have to learn just a few concepts and commands. It uses the formula language that is also used in almost all of the statistical modeling functions. This commonality is a win. You can customize plots, but it is slightly less obvious how as we interact with a) panel functions, and b) the grid graphics system.

Many people take the time to learn ggplot2 and is probably a good idea if you are using R as your primary tool for visualizing data. It produces all of the common plots, and provides a computational framework for describing what is to be plotted using Wilkinson's Grammar of Graphics. The learning curve for ggplot2 is slightly higher than the other two, but not much. ggplot2 requires data frames, and not everything naturally fits in a data frame, e.g. two samples with different numbers of observations.

The lines, points, text are rendered using one of two graphics systems

  • traditional graphics engine

  • grid - used by lattice and ggplot2

grid is much more sophisticated, flexible and powerful, but there is a learning curve and a reasonable amount to remember. The traditional graphics gets you a long way. You don't need to learn grid to use ggplot2 or lattice. 

d = read.csv("gst_file.csv", stringsAsFactors = FALSE)

plot(-1* d$LONGITUDE, d$LATITUDE)

plot(-1* d$LONGITUDE, d$LATITUDE, pch = ".")
Add proper labels for the horizontal and vertical axes. 
plot(-1* d$LONGITUDE, d$LATITUDE, pch = ".", xlab = "Longitude", ylab = "Latitude")

map('county', xlim = c(-125, -113), ylim = c(30, 44))
points(-1* d$LONGITUDE, d$LATITUDE, pch = ".", col = "red")

Lattice

The basic idea is that we have a data frame containing observations with several variables. We can use it with one or two variables for univariate or bivariate plots. But it is more interesting when we

  1. group observations within a single plot

  2. create multiple panels/plots for different subsets of the original data

  3. group observations within multiple panels/plots for different subsets of the original data

One benefit of lattice is that it uses R's formula notation that is used in almost all the modeling functions in R. The basic format in lattice is 

plotFunc( y ~ x | panelGroupVariable, data)
